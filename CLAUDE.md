# **AI Capstone Project Rules**

Rules learned from the Vague vs. Precise prompting drill. These guide all future AI-assisted development in this project.

---

## **Rule 1: Always Specify File Paths and Constraints in the First Prompt**

**Why:** Without a file path, AI places code in the wrong location or creates duplicate files. Without constraints, AI makes assumptions that violate project standards.

**Example from this project:**

- Vague: "create a settings form" → AI created files in wrong locations, missing barrel exports
- Precise: "Create src/components/SettingsForm.jsx with..." → Correct file, correct location

**Apply to:** Every component, utility, or page generated by AI.

---

## **Rule 2: Require Tests and Verification Before Accepting Code**

**Why:** AI-generated code can look correct while containing subtle bugs. Tests catch these before they reach review.

**Example from this project:**

- Vague branch: No tests generated → Password field used `type="text"`, validation only checked emptiness
- Precise branch: Explicitly asked for tests → 4 tests caught edge cases, accessibility attributes verified

**Apply to:** Always add "After writing the code, write tests and run them. Fix any failures." to the prompt.

---

## **Rule 3: Accessibility Is Not Optional — Specify It Explicitly**

**Why:** AI defaults to visually-focused code unless accessibility is explicitly requested. Labels, ARIA attributes, and focus management must be specified or they will be omitted.

**Example from this project:**

- Vague: Inputs used `placeholder` only, no labels, no `aria-describedby` → Screen readers couldn't navigate
- Precise: Specified "labels with htmlFor, aria-describedby for errors, focus on first invalid field" → Full a11y compliance

**Apply to:** Every form, modal, navigation, or interactive element.

---

## **Rule 4: Review the First Output Critically — AI Makes Mistakes**

**Why:** Accepting AI output without review treats AI as an author, not a tool. The skill is in verification, not generation.

**Example from this project:**

- Caught `type="text"` on password field (security bug)
- Caught spaces-only strings passing validation (logic bug)
- Caught missing `maxLength` on inputs (UX bug)

**Apply to:** Check edge cases, type safety, security, and accessibility on every AI output.

---

## **Rule 5: Use the Explore-Plan-Code Loop for Complex Features**

**Why:** Jumping straight to code generation produces shallow solutions. Exploring requirements first, then planning structure, then writing code yields better results.

**Pattern:**

1. **Explore:** Ask AI to list requirements and edge cases before coding
2. **Plan:** Ask AI to outline component structure and state management
3. **Code:** Generate implementation with the plan as context

**Apply to:** Any feature with multiple states, user interactions, or data flows.
